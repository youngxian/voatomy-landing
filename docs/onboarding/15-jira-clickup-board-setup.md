# Board Setup Guide: Jira and ClickUp (for ATLAS)

This guide helps teams that do not have a sprint board yet, or do not want to use sprints today, get into a board structure that ATLAS can learn and plan from.

Principles:
- Keep it simple. A usable board beats a perfect board.
- ATLAS can plan without a board (Start Fresh), but boards unlock better learning and reports.

---

## When to Use This Guide

Use this guide if:
- The team does not have Jira/ClickUp yet.
- The team has Jira/ClickUp but no sprint board.
- The team does not run sprints but wants lightweight planning and reporting.

---

## Minimal Board Requirements (Tool-Agnostic)

ATLAS works best when the board has:
- Statuses: Backlog, Ready, In Progress, Done
- Fields: Assignee, Priority (optional), Points (optional), Sprint/Cycle (optional)
- Work classes: Feature, Bug, Chore (or labels that approximate these)

If points are not used:
- ATLAS can still plan using complexity and capacity signals.
- Points can be introduced later.

---

## Jira: Quick Setup Checklist

Create:
- One project per product area (or one project to start).
- One board:
  - Scrum board if you run sprints.
  - Kanban board if you prefer continuous flow.

Statuses:
- Backlog
- Selected for Development (or Ready)
- In Progress
- In Review (optional)
- Done

Sprint setup (Scrum):
- Create the current sprint if you are already mid-sprint.
- Otherwise create “Sprint 1” and keep it empty until ATLAS proposes a plan.

Recommended permissions:
- Read-only access for ATLAS initially.
- Optional: enable commenting later if you want ATLAS to annotate planned tickets.

## ClickUp: Quick Setup Checklist

Create:
- Workspace
- Space (Product)
- Folder (Project)
- List (Backlog)
- Optional List (Sprint)

Statuses:
- Backlog
- Ready
- In Progress
- Blocked (optional)
- Done

Sprints in ClickUp:
- If you already run sprints, enable the sprint feature and create the current sprint.
- If you don’t run sprints, keep it Kanban-style and let ATLAS plan in a rolling window.

## If the Team Is Not Interested in Sprints

ATLAS supports “continuous planning”:
- Use a Kanban board.
- Plan a rolling 7–14 day window.
- Generate weekly reports and a goal for the week.

This is still sprint-like value without sprint ceremony.
