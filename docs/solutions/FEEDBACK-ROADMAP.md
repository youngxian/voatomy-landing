# Use Case: Feedback to Roadmap
## by Voatomy Labs

> **"Customer demand directly into sprint prioritization"**

---

## Problem
Customer feedback is scattered across Slack threads, support tickets, sales calls, and spreadsheets. By the time it reaches engineering, it's stale, decontextualized, and missing revenue context. Teams build features nobody asked for while ignoring the ones blocking real pipeline.

## Solution
LOOP captures customer signals from every source, weights them by revenue impact, and feeds them directly into the product backlog — creating a closed loop from feedback to ship to customer notification.

## Primary Product
**LOOP** (#6366F1) — Revenue Feedback Engine

## The Broken Funnel → Fixed
| Source | Before (Broken) | After (LOOP) |
|--------|-----------------|--------------|
| Sales calls | → Slack thread → Forgotten | → AI extraction → Revenue-weighted backlog |
| Support tickets | → Spreadsheet → Outdated | → Signal aggregation → Demand scoring |
| Churn signals | → Quarterly review → Too late | → Real-time detection → Priority flag |
| NPS surveys | → PDF report → Never read | → Theme clustering → Feature mapping |

## The Complete Pipeline
1. **Capture**: Gong, Zendesk, Salesforce, NPS
2. **Process**: AI aggregates, deduplicates, weights by revenue
3. **Prioritize**: Revenue-weighted backlog with dollar values
4. **Ship**: Feature delivered in sprint
5. **Close Loop**: Auto-notify requesting accounts

## Example Feature Journey
- "SSO Authentication" requested by 47 accounts = $2.3M pipeline
- Prioritized in Sprint 26, shipped in Sprint 28
- Sales notified → 12 deals accelerated → $1.4M influenced revenue
